🎯 1. 系统核心理念

    不可变时间线 (Immutable Timeline)： 记录即定格。备忘录以 Commit 形式存在，仅包含文本和时间戳，一经提交不可修改，强制记录最真实的思维演进逻辑。

    低阻力捕获 (Low-Friction Capture)： 抛弃繁琐的点击与模式切换。呼出系统、直接打字、回车定格，保持心流不断。

    单线聚焦 (Single Branch)： 摒弃复杂的分支与合并，每个“系列（Series）”只维护一条纯粹的、按时间倒序排列的 Commit 时间线。

⌨️ 2. 键鼠交互与全局操作定义

系统支持纯键盘极速操作，同时兼容鼠标点选作为辅助。

    全局唤醒： 自定义全局热键（如 Alt + Space 或 Cmd + Space）一键呼出/隐藏主界面。

    主界面导航（第一级）：

        ↑ / ↓ 键：在各个系列之间上下切换高亮状态。

        鼠标：支持滚轮上下滚动，左键单击高亮系列。

    新建与搜索：

        新建系列： 按下 Shift + N，触发新建系列弹窗/输入框。

        搜索过滤： 按下 / 键，呼出搜索框，直接打字过滤系列名称；按 Esc 退出搜索。

    内容录入（核心动作）：

        在第一级界面，高亮某个系列时，直接敲击键盘任意字符即自动聚焦到底部的输入框，开始撰写新 Commit。

        按下 Enter 键：保存并提交该 Commit，系统自动更新该系列的最顶层展示。

    层级穿梭：

        进入第二级： 按下 →（右方向键）或鼠标双击当前高亮的系列，进入该系列的完整 Commit 时间线视图。

        返回第一级： 按下 ←（左方向键）或 Esc 键，返回系列列表视图。

👁️ 3. 两级视图视觉与业务逻辑
第一级：系列列表视图 (Series List)

    信息密度： 列表项仅显示“系列名称” + “最新一次 Commit 的文本摘录”。

    动态排序： 列表严格按照“最后更新时间”倒序排列，刚刚追加过 Commit 的系列永远在最顶部。

    沉寂与归档机制：

        沉寂判定： 超过 7 天没有新增 Commit 的系列，自动沉寂。

        视觉提醒： 沉寂系列会被自动沉底，并使用特殊的暗色调或特定标记显示。

        快速归档： 光标高亮沉寂系列时，按下 a 键，直接将其移出主列表，放入归档数据库。

第二级：Commit 时间线视图 (Timeline)

    视觉呈现： 从上到下，按时间倒序（最新的在最上面）排列的单线视图。

    元素展示： 每个节点只包含 Commit 正文文本和精确到秒的时间戳。此界面仅供回顾，不提供“修改”按钮。

💾 4. 双轨数据架构设计

为了满足你学习 PostgreSQL 的需求，同时保证应用的基础可用性，系统底层将采用仓储模式 (Repository Pattern) 进行抽象。

    双重实现： 在代码的持久层（Data Access Layer），你需要定义一个统一的接口（例如 MemoRepository），然后分别实现 SQLiteRepository 和 PostgresRepository。

    配置驱动： 在配置文件或环境变量中提供一个开关（如 DB_ENGINE=sqlite|postgres 或 ENABLE_POSTGRES=true）。

    学习期双写（推荐）： 在项目初期，你可以尝试实现“双写”逻辑——每次按下 Enter 提交 Commit 时，系统同时向本地的 SQLite 文件和远端的 PostgreSQL 写入数据。这样即便你配置 PostgreSQL 时出错，本地数据也绝对安全。后期学成后，可以通过配置彻底关闭或剥离 PostgreSQL。
