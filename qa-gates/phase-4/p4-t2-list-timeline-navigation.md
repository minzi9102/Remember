# P4-T2 主轨道与时间线十字导航 门禁测试方案

## 目标
- 验证 Cross-Axis vNext 的主副轴导航一致性。
- 本文件聚焦：`←/→` 主轨道切换、`↓` 进入 Timeline、`Esc` 返回主轨道。

## 前置条件
1. 已获取本版本可执行程序与启动命令。
2. 可访问目标配置文件（`config.toml`）并允许重启应用。
3. 可查看应用日志目录；如涉及一致性验证，可访问 SQLite 查询工具。
4. 所有步骤按黑盒执行，不阅读源码。

## 测试数据
| 数据项 | 建议值 |
|---|---|
| 运行模式 | sqlite_only |
| 样例 Series | Inbox, Project-A, Archive-Me |
| 样例 Commit | first-note, follow-up-note, rollback-check |
| 异常输入 | 空字符串、超长文本(2000字) |

## 视觉门禁（2例）
### P4-T2-VG-PASS
- 用例目的：验证十字架构导航在正常路径下可见且稳定。
- 执行步骤：
  1. 启动应用并进入 Active 主轨道。
  2. 观察横向 Series 轨道与当前高亮卡片状态。
  3. 按 `↓` 进入 Timeline，确认在当前卡片下方纵向展开。
  4. 按 `Esc` 返回主轨道并截图关键界面。
- 预期结果：
  1. 主轨道横向布局与 Timeline 纵向布局层次清晰。
  2. 进入/返回过程无错位、无异常闪烁。
- 必交证据：`P4-T2-VG-PASS_*.png` + `P4-T2-VG-PASS_*.txt`

### P4-T2-VG-FAIL
- 用例目的：验证异常导航路径下有明确保护反馈。
- 执行步骤：
  1. 在无选中项或加载异常场景触发 `↓`。
  2. 在 Timeline 未打开场景触发 `Esc`。
  3. 观察界面是否出现错误提示或安全无动作。
- 预期结果：
  1. 不发生错误视图跳转或崩溃。
  2. 有可追踪的失败提示或一致的无动作策略。
- 必交证据：`P4-T2-VG-FAIL_*.png` + `P4-T2-VG-FAIL_*.txt`

## 交互门禁（2例）
### P4-T2-IG-PASS
- 用例目的：验证十字导航主路径形成闭环。
- 执行步骤：
  1. 在主轨道使用 `←/→` 切换高亮 Series。
  2. 对当前高亮项按 `↓` 进入 Timeline。
  3. 按 `Esc` 返回主轨道，确认焦点回到原 Series。
- 预期结果：
  1. 主轨道与 Timeline 的切换路径稳定可重复。
  2. 焦点回退行为一致，不丢失上下文。
- 必交证据：`P4-T2-IG-PASS_*.mp4` + `P4-T2-IG-PASS_*.txt`

### P4-T2-IG-FAIL
- 用例目的：验证异常交互被正确拦截且系统可恢复。
- 执行步骤：
  1. 连续快速触发 `↓` 与 `Esc` 制造边界冲突。
  2. 记录系统反馈与恢复路径。
  3. 再执行一次标准 `←/→ -> ↓ -> Esc` 链路。
- 预期结果：
  1. 边界冲突不导致卡死或不可恢复状态。
  2. 后续合法导航可继续完成。
- 必交证据：`P4-T2-IG-FAIL_*.mp4` + `P4-T2-IG-FAIL_*.txt`

## 证据要求
1. 命名遵循 `qa-gates/EVIDENCE-NAMING.md`。
2. 每条用例至少提交 1 份视觉证据 + 1 份交互证据。
3. 文本证据需包含：执行环境、步骤编号、实际结果、结论。
4. 涉及数据一致性时，附 SQLite 查询截图或导出文本。

## 通过标准
1. `P4-T2-VG-PASS` 与 `P4-T2-IG-PASS` 均通过。
2. `P4-T2-VG-FAIL` 与 `P4-T2-IG-FAIL` 能触发预期保护。
3. 证据完整可追溯，结论可被第三方复验。

## 失败分级
| 等级 | 判定标准 | 示例 |
|---|---|---|
| 阻断 | 主路径不可用或一致性破坏 | 无法进入 Timeline、返回后焦点丢失、应用崩溃 |
| 高 | 核心行为错误但可临时绕过 | 方向键切换错乱、视图状态不同步 |
| 中 | 非核心流程偏差 | 提示文案不一致、状态提示延迟 |
| 低 | 轻微视觉问题 | 对齐偏差、间距异常 |
