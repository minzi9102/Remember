# P4-T3 键盘优先交互（Cross-Axis）门禁测试方案

## 目标
- 验证 vNext 键盘契约在不同焦点场景下行为稳定。
- 本文件聚焦：主轨道、Timeline、全局浮层、附着草稿的全键盘流。

## 前置条件
1. 已获取本版本可执行程序与启动命令。
2. 可访问目标配置文件（`config.toml`）并允许重启应用。
3. 可查看应用日志目录；如涉及一致性验证，可访问 SQLite 查询工具。
4. 所有步骤按黑盒执行，不阅读源码。

## 测试数据
| 数据项 | 建议值 |
|---|---|
| 运行模式 | sqlite_only |
| 样例 Series | Inbox, Project-A, Archive-Me |
| 样例 Commit | first-note, follow-up-note, rollback-check |
| 键位集合 | ←/→/↓/↑/Esc//Shift+N/Enter/a |

## 视觉门禁（2例）
### P4-T3-VG-PASS
- 用例目的：验证关键键位触发的 UI 层级与状态反馈正确可见。
- 执行步骤：
  1. 在主轨道执行 `←/→` 并观察高亮变化。
  2. 按 `/` 打开全局搜索条，按 `Esc` 关闭。
  3. 按 `Shift+N` 打开全局新建条，按 `Esc` 关闭。
  4. 直接输入字符触发附着草稿框并截图。
- 预期结果：
  1. 全局浮层与附着草稿框视觉层级清晰。
  2. 各键位触发行为与文案提示一致。
- 必交证据：`P4-T3-VG-PASS_*.png` + `P4-T3-VG-PASS_*.txt`

### P4-T3-VG-FAIL
- 用例目的：验证无效或冲突按键不会造成错误副作用。
- 执行步骤：
  1. 在无可写上下文输入非法快捷组合。
  2. 在非沉寂 Series 上触发 `a`。
  3. 在归档集合中尝试触发写操作按键。
- 预期结果：
  1. 非法交互被拦截并有明确反馈。
  2. 页面不崩溃，后续仍可执行合法按键。
- 必交证据：`P4-T3-VG-FAIL_*.png` + `P4-T3-VG-FAIL_*.txt`

## 交互门禁（2例）
### P4-T3-IG-PASS
- 用例目的：验证完整键盘主路径可闭环。
- 执行步骤：
  1. `←/→` 选择目标 Series。
  2. 直接输入字符触发附着草稿，按 `Enter` 提交。
  3. 按 `↓` 进入 Timeline，使用 `↑/↓` 浏览历史，再按 `Esc` 返回。
  4. 使用 `/` 搜索并恢复上下文。
- 预期结果：
  1. 键盘链路完整，无卡死、无误触发。
  2. 提交、导航、搜索结果可被 UI 与日志/查询交叉佐证。
- 必交证据：`P4-T3-IG-PASS_*.mp4` + `P4-T3-IG-PASS_*.txt`

### P4-T3-IG-FAIL
- 用例目的：验证反向交互被正确拦截且系统可恢复。
- 执行步骤：
  1. 在归档集合中依次尝试直接输入、`Shift+N`、`a`。
  2. 在主轨道无选中项时尝试直接输入。
  3. 执行一次合法键盘路径确认恢复能力。
- 预期结果：
  1. 被禁止操作返回明确错误提示，不得静默失败。
  2. 合法链路仍可正常完成。
- 必交证据：`P4-T3-IG-FAIL_*.mp4` + `P4-T3-IG-FAIL_*.txt`

## 证据要求
1. 命名遵循 `qa-gates/EVIDENCE-NAMING.md`。
2. 每条用例至少提交 1 份视觉证据 + 1 份交互证据。
3. 文本证据需包含：执行环境、步骤编号、实际结果、结论。
4. 涉及数据一致性时，附 SQLite 查询截图或导出文本。

## 通过标准
1. `P4-T3-VG-PASS` 与 `P4-T3-IG-PASS` 均通过。
2. `P4-T3-VG-FAIL` 与 `P4-T3-IG-FAIL` 能触发预期保护。
3. 证据完整可追溯，结论可被第三方复验。

## 失败分级
| 等级 | 判定标准 | 示例 |
|---|---|---|
| 阻断 | 主路径不可用或一致性破坏 | 键盘无法提交、无法返回主轨道、应用崩溃 |
| 高 | 核心行为错误但可临时绕过 | 键位映射错乱、焦点状态异常 |
| 中 | 非核心流程偏差 | 提示文案不一致、边界状态遗漏 |
| 低 | 轻微视觉问题 | 提示层位置偏差、间距异常 |
