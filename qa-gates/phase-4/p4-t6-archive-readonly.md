# P4-T6 归档切换与只读时间线 门禁测试方案

## 目标
- 验证归档后从 Active 主轨道移除，并在 Archived 集合保持只读。
- 本文件聚焦：`Archived` 按钮点击切换策略与归档 Timeline 只读约束。

## 前置条件
1. 已获取本版本可执行程序与启动命令。
2. 可访问目标配置文件（`config.toml`）并允许重启应用。
3. 可查看应用日志目录；如涉及一致性验证，可访问 SQLite 查询工具。
4. 所有步骤按黑盒执行，不阅读源码。

## 测试数据
| 数据项 | 建议值 |
|---|---|
| 运行模式 | sqlite_only |
| 样例 Series | Inbox, Project-A, Archive-Me |
| 样例 Commit | first-note, follow-up-note, rollback-check |
| 异常输入 | 归档后输入尝试、空归档集合 |

## 视觉门禁（2例）
### P4-T6-VG-PASS
- 用例目的：验证归档后只读与按钮切换在正常路径可见。
- 执行步骤：
  1. 在 Active 集合归档一个 `silent` Series。
  2. 确认该项从 Active 主轨道移除。
  3. 点击右上 `Archived` 按钮进入归档集合并截图。
  4. 打开归档 Timeline，核对只读提示。
- 预期结果：
  1. `Archived` 按钮始终可见且点击可切换集合。
  2. 归档 Timeline 仅展示历史，不出现可编辑入口。
- 必交证据：`P4-T6-VG-PASS_*.png` + `P4-T6-VG-PASS_*.txt`

### P4-T6-VG-FAIL
- 用例目的：验证归档只读被破坏时有明确保护。
- 执行步骤：
  1. 在归档集合中尝试触发写入入口。
  2. 观察是否出现非法可编辑控件或错误状态。
  3. 记录失败反馈。
- 预期结果：
  1. 所有写路径均被禁止并反馈可读错误。
  2. 界面不崩溃，归档内容仍可回看。
- 必交证据：`P4-T6-VG-FAIL_*.png` + `P4-T6-VG-FAIL_*.txt`

## 交互门禁（2例）
### P4-T6-IG-PASS
- 用例目的：验证归档与只读回看形成闭环。
- 执行步骤：
  1. 在 Active 集合高亮沉寂项并按 `a` 归档。
  2. 点击右上 `Archived` 按钮切换集合。
  3. 进入归档 Timeline，按 `Esc` 返回归档主轨道。
- 预期结果：
  1. 归档动作与集合切换链路稳定可执行。
  2. Timeline 回看只读且可正常返回。
- 必交证据：`P4-T6-IG-PASS_*.mp4` + `P4-T6-IG-PASS_*.txt`

### P4-T6-IG-FAIL
- 用例目的：验证反向交互被正确拦截。
- 执行步骤：
  1. 在归档集合尝试直接输入、`Shift+N`、`a`。
  2. 尝试通过键盘快捷方式切换集合（不点击按钮）。
  3. 再点击 `Archived` 或 `Active` 按钮验证系统可恢复。
- 预期结果：
  1. 归档集合写操作全部被拒绝并反馈明确。
  2. 集合切换策略保持 `mouse-only`，键盘路径不应绕过。
- 必交证据：`P4-T6-IG-FAIL_*.mp4` + `P4-T6-IG-FAIL_*.txt`

## 证据要求
1. 命名遵循 `qa-gates/EVIDENCE-NAMING.md`。
2. 每条用例至少提交 1 份视觉证据 + 1 份交互证据。

4. 涉及数据一致性时，附 SQLite 查询截图或导出文本。

## 通过标准
1. `P4-T6-VG-PASS` 与 `P4-T6-IG-PASS` 均通过。
2. `P4-T6-VG-FAIL` 与 `P4-T6-IG-FAIL` 能触发预期保护。
3. 证据完整可追溯，结论可被第三方复验。

## 失败分级
| 等级 | 判定标准 | 示例 |
|---|---|---|
| 阻断 | 主路径不可用或一致性破坏 | 归档后仍在 Active、归档可写、应用崩溃 |
| 高 | 核心行为错误但可临时绕过 | 归档切换异常、只读约束失效 |
| 中 | 非核心流程偏差 | 提示文案不一致、返回状态异常 |
| 低 | 轻微视觉问题 | 徽标样式偏差 |
