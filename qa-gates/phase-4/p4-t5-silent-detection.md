# P4-T5 沉寂识别与主轨道下沉 门禁测试方案

## 目标
- 验证沉寂 Series 自动标记并在主轨道中下沉显示。
- 本文件聚焦：沉寂判定、标记稳定性与主轨道上下文一致性。

## 前置条件
1. 已获取本版本可执行程序与启动命令。
2. 可访问目标配置文件（`config.toml`）并允许重启应用。
3. 可查看应用日志目录；如涉及一致性验证，可访问 SQLite 查询工具。
4. 所有步骤按黑盒执行，不阅读源码。

## 测试数据
| 数据项 | 建议值 |
|---|---|
| 运行模式 | sqlite_only |
| 样例 Series | Inbox, Project-A, Archive-Me |
| 样例 Commit | first-note, follow-up-note, rollback-check |
| 异常输入 | 阈值边界时间、空列表、超长文本(2000字) |

## 视觉门禁（2例）
### P4-T5-VG-PASS
- 用例目的：验证沉寂下沉在正常路径下可见且稳定。
- 执行步骤：
  1. 触发沉寂扫描并刷新主轨道。
  2. 观察沉寂 Series 的标记与位置变化。
  3. 截图操作前后主轨道状态。
- 预期结果：
  1. 超过阈值项显示 `silent` 状态标记。
  2. 沉寂项在主轨道中下沉，不占据最新最左区域。
- 必交证据：`P4-T5-VG-PASS_*.png` + `P4-T5-VG-PASS_*.txt`

### P4-T5-VG-FAIL
- 用例目的：验证误判或阈值异常时有明确保护。
- 执行步骤：
  1. 使用阈值边界数据触发扫描。
  2. 观察是否出现误标或位置异常。
  3. 记录失败提示与界面回退。
- 预期结果：
  1. 误判场景可被识别并可追踪。
  2. 界面不崩溃，列表可继续操作。
- 必交证据：`P4-T5-VG-FAIL_*.png` + `P4-T5-VG-FAIL_*.txt`

## 交互门禁（2例）
### P4-T5-IG-PASS
- 用例目的：验证沉寂识别链路与后续操作可闭环。
- 执行步骤：
  1. 触发沉寂扫描并确认目标项变为 `silent`。
  2. 对该项追加新 Commit 或执行归档路径。
  3. 验证状态更新与主轨道位置同步变化。
- 预期结果：
  1. 扫描结果稳定且可复验。
  2. 状态变化与轨道展示一致。
- 必交证据：`P4-T5-IG-PASS_*.mp4` + `P4-T5-IG-PASS_*.txt`

### P4-T5-IG-FAIL
- 用例目的：验证反向交互被拦截且系统可恢复。
- 执行步骤：
  1. 在不满足条件场景触发沉寂相关操作。
  2. 观察拦截反馈与恢复路径。
  3. 再执行一次合法扫描确认恢复。
- 预期结果：
  1. 非法交互被拒绝并有明确提示。
  2. 合法操作链路可继续完成。
- 必交证据：`P4-T5-IG-FAIL_*.mp4` + `P4-T5-IG-FAIL_*.txt`

## 证据要求
1. 命名遵循 `qa-gates/EVIDENCE-NAMING.md`。
2. 每条用例至少提交 1 份视觉证据 + 1 份交互证据。
3. 文本证据需包含：执行环境、步骤编号、实际结果、结论。
4. 涉及数据一致性时，附 SQLite 查询截图或导出文本。

## 通过标准
1. `P4-T5-VG-PASS` 与 `P4-T5-IG-PASS` 均通过。
2. `P4-T5-VG-FAIL` 与 `P4-T5-IG-FAIL` 能触发预期保护。
3. 证据完整可追溯，结论可被第三方复验。

## 失败分级
| 等级 | 判定标准 | 示例 |
|---|---|---|
| 阻断 | 主路径不可用或一致性破坏 | 沉寂判定失效、应用崩溃 |
| 高 | 核心行为错误但可临时绕过 | 沉寂标记错误、位置错乱 |
| 中 | 非核心流程偏差 | 提示文案不一致、刷新延迟 |
| 低 | 轻微视觉问题 | 标签样式偏差 |
